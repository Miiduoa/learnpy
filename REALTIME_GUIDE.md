# 實時引導系統說明

## 概述

實時引導系統在學習者輸入程式碼的過程中提供智能提示和引導，幫助學習者更好地理解題目要求並寫出正確的程式碼。

## 設計理念

### 1. **非侵入式設計**
- 提示以輕量級、不干擾的方式顯示
- 不會打斷學習者的思考流程
- 可以自動隱藏，避免視覺干擾

### 2. **分層引導**
系統提供三種幫助級別：

#### **最小幫助 (Minimal)**
- 只在有嚴重錯誤時提示
- 延遲時間較長（5秒）
- 適合有經驗的學習者

#### **適中幫助 (Moderate)** - 預設
- 顯示重要提示和警告
- 延遲時間適中（2秒）
- 適合大多數學習者

#### **完整幫助 (Helpful)**
- 顯示所有提示和建議
- 延遲時間較短（1秒）
- 適合初學者

### 3. **智能判斷**
- 根據課程要求提供上下文相關的提示
- 檢測常見錯誤（拼寫錯誤、縮排問題等）
- 識別硬編碼模式並建議使用循環

## 功能特點

### 1. **實時語法檢查**
- 檢查縮排問題（例如：冒號後需要縮排）
- 檢測常見拼寫錯誤（prnt → print, rng → range 等）
- 提供即時反饋

### 2. **上下文相關提示**
- 根據當前輸入內容提供建議
- 例如：輸入 `for` 時提示使用 `range()`
- 輸入 `def` 時提示需要冒號

### 3. **課程要求引導**
- 檢查是否使用了要求的程式結構（循環、函數等）
- 檢測是否違反了禁止項（如硬編碼）
- 提供符合題目要求的建議

### 4. **漸進式提示**
- 不會在輸入開始時立即提示
- 等待學習者思考一段時間後再提供幫助
- 避免過度依賴提示

## 使用方式

### 自動啟用
系統會在載入課程時自動初始化實時引導系統。

### 手動設置幫助級別
```javascript
// 在瀏覽器控制台執行
localStorage.setItem('guideLevel', 'minimal'); // 或 'moderate', 'helpful'
location.reload(); // 重新載入頁面以應用設置
```

### 程式化控制
```javascript
// 設置幫助級別
realtimeGuide.setGuideLevel('moderate');

// 清除所有提示
realtimeGuide.clear();
```

## 提示類型

### 1. **語法錯誤 (Error)**
- 紅色邊框
- 立即顯示
- 例如：縮排錯誤、語法錯誤

### 2. **警告 (Warning)**
- 橙色邊框
- 例如：檢測到硬編碼

### 3. **信息提示 (Info)**
- 藍色邊框
- 例如：缺少要求的程式結構

### 4. **輕微提示 (Hint)**
- 灰色邊框
- 例如：拼寫建議、上下文提示

## 最佳實踐

### 對學習者的建議

1. **先思考，再看提示**
   - 嘗試自己解決問題
   - 提示是輔助工具，不是答案

2. **理解提示的意義**
   - 不要只是照著提示修改
   - 理解為什麼需要這樣寫

3. **根據需要調整幫助級別**
   - 初學者可以使用「完整幫助」
   - 有經驗後可以切換到「最小幫助」

### 對教師的建議

1. **在課程中明確要求**
   - 在 `code_requirements` 中明確指定要求
   - 這樣系統才能提供準確的引導

2. **平衡提示和挑戰**
   - 不要讓提示過於詳細
   - 保留一些思考空間給學習者

3. **觀察學習者的反應**
   - 如果學習者過度依賴提示，建議降低幫助級別
   - 如果學習者經常卡住，可以增加幫助級別

## 技術實現

### 前端實現
- `realtime-guide.js` - 實時引導系統核心
- 使用事件監聽器監聽輸入
- 輕量級語法檢查（不依賴完整解析）

### 後端支援

## 常見問題

### Q: 提示會影響學習效果嗎？
A: 如果使用得當，實時引導可以幫助學習者更快理解概念。關鍵是：
- 使用適當的幫助級別
- 不要過度依賴提示
- 理解提示背後的原理

### Q: 如何關閉實時引導？
A: 設置幫助級別為 'minimal'，這樣只會在嚴重錯誤時提示。

### Q: 提示會顯示答案嗎？
A: 不會。提示只會引導學習者使用正確的方法，不會直接給出答案。

### Q: 可以自定義提示內容嗎？
A: 目前提示內容是根據課程的 `code_requirements` 自動生成的。未來可能會支援自定義提示。
